# D4 数据集与图中表现、以及「为什么是线性」说明

## 1. D4 在你这份数据里长什么样

从 **paper1_results_v3/latent_and_gold_all.csv** 里对 D4 的逐行查看可以归纳出：

- **D4 在这份数据里是有 SSPG/DI 的**  
  配置里虽然写的是 `has_sspg=False, has_di=False`（Hall 2018 原文没有 clamp），说明你当前用的数据包（例如 cgm_all_datasets 或合并后的 labels）里，**给 D4 配了金标准**，所以图里会出现绿色的 D4 点。

- **D4 的 DI 几乎都挤在很小的范围**  
  - D1/D2：DI 常见在 0.4～3、甚至更高。  
  - D4：绝大多数在 **0.01～0.15** 之间（如 0.06, 0.039, 0.068, 0.089, 0.037, 0.016, 0.053, 0.112, 0.086…），只有极少数略高。  
  所以从数值上看，D4 的「金标准 DI」整体非常低，且**分布很窄**。

- **D4 的 SSPG**  
  范围大约从几十到三百多（和 D1/D2 有重叠），但样本量小（约 20 个有金标准的 subject），容易在图上形成一小团或一条带。

- **没有对 D4 单独调权重**  
  画图时：所有点（D1/D2/D4）一视同仁，按 `dataset_id` 着色；**没有**对 D4 提高或降低权重。  
  评估时：Ridge 也是在全表上（或 5-fold 内）一起拟合，**没有**按 dataset 加权。  
  所以图里 D4「长成那样」是因为**数据本身**的分布，不是我们故意压或抬 D4。

## 2. 图里 D4 为什么看起来「怪」——可能原因

- **(a) 人的组成 / 协议**  
  D4 来自 Hall et al. 2018，自由生活 CGM。若你这批 D4 的金标准是后来从别处合并的（例如另一批做了 clamp 的受试者），很可能：  
  - 人群不同（例如更多已确诊、β 细胞功能差），和  
  - 测定方式/定义不同（例如 DI 单位或公式不一致）。  
  那样的话，**确实是「人的问题」**：D4 代表的是另一类人群或另一种测量，和 D1/D2 不在同一把尺子上。

- **(b) 分布本身的特点**  
  - DI：D4 的 DI 几乎全挤在 0.01～0.15，**方差极小**。  
  - 在「mi vs DI」图里：横轴（DI）在 D4 上几乎只有一小段，纵轴（mi）仍有一定跨度 → 绿色点会形成**近似一条竖带或斜带**，看起来和 D1/D2 的「一大片」很不一样。  
  - 在留一数据集出、用 D1+D2 训 Ridge 再在 D4 上测时：D4 的 DI 范围与 D1/D2 差异大，Ridge 外推会很不稳，容易出现极端 RMSE/R²（例如 V3 里 D4 的 SSPG 留一数据集出崩掉）。

- **(c) 是否「大部分是健康人」**  
  从现有 CSV 看，D4 的 **DI 普遍很低**，更像是**β 细胞功能偏弱/受损**的一侧，而不是「健康人为主」。所以更合理的解读是：D4 要么是不同人群/不同定义，要么是测量或合并方式导致 DI 整体被压得很低且范围窄。  
  若将来你确认 D4 的 labels 来源（是否同协议、同定义），可以更准确说是「人的问题」还是「数据/定义问题」。

- **小结**  
  - 图里 D4 的样子 = **数据分布**（DI 极窄、SSPG 样本少） + **没有对 D4 单独加权**。  
  - 留一数据集出在 D4 上崩 = **分布/尺度与 D1+D2 不一致**，Ridge 线性外推不稳。  
  - 算法没有针对 D4 做特殊权重；若希望「不拖累主结果」，当前做法就是：**主结果只用 D1+D2 评估（如 V4）**，D4 仅作补充或表型。

## 3. 我们怎么知道回归该用线性？——其实**不知道**，只是选了线性并做了对比

- **事实**：潜变量（si, mi, …）与金标准 SSPG/DI 的**真实关系我们并不知道**。在 Bergman 模型里，稳态血糖与 SI 的关系来自 ODE 解，本身是**非线性的**；DI 常定义为 SI × 分泌相关量，也可能是乘积等非线性关系。所以**没有先验理由假定线性**。

- **当前做法**：我们**默认**用了 **Ridge（线性）**，原因是：(1) 样本量小（n≈63），线性更稳、可解释；(2) 没有先做系统对比就沿用了常见做法。这是**工程/习惯选择**，不是「验证过关系是线性的」。

- **已做的对比**（`scripts/compare_linear_vs_nonlinear.py`）：在**相同 5-fold（按 subject）**下对比：
  - **Ridge 线性**（当前主方法）
  - **Ridge + 对 log(SSPG+1) / log(DI+ε) 回归再反变换**
  - **二次项 + Ridge**（PolynomialFeatures degree=2）
  - **GradientBoostingRegressor**（轻度非线性）
  
  在 V4 的 D1+D2 数据上跑出的结果示例：
  - **SSPG**：Ridge 线性 r≈0.63、RMSE≈56；log-target / poly2 / GB 的 r 均**低于**线性，RMSE 相当或更差。
  - **DI**：Ridge 线性 r≈0.39；其余三种均**低于**线性。

  - **仅 D1+D2（V4）**：线性 Ridge 在 SSPG 和 DI 上都是最佳或相当，非线性未占优。  
  - **含 D4（V3，D1+D2+D4）**：SSPG 仍以线性最佳（r≈0.52）；**DI 上二次项 (Ridge_poly2) 明显优于线性**（Spearman r≈0.70 vs 0.63），说明在纳入 D4 后，潜变量与 DI 的关系更接近非线性。详见 `paper1_results_v3/linear_vs_nonlinear.csv`。  

  结论：我们**不假定**关系为线性；在仅 D1+D2 时线性足够，**在含 D4 时 DI 建议用二次项或报告线性+非线性对比**。

- **两处「线性」在代码里**  
  1. **评估/预测**：`evaluate_p1_metrics.py` 里用 **Ridge(6D → target)**，即 y = Xw + b。  
  2. **图里的线**：`plot_p1_results.py` 里用 `np.polyfit(..., 1)` 画**一条直线**作趋势，仅用于可视化概括，不是「假定真实关系是直线」。

## 4. 关于 D3

- **D3（CGMacros 等）** 在配置中 **没有 SSPG/DI 金标准**（`has_sspg=False, has_di=False`），因此**无法**参与「预测 SSPG/DI」的评估或线性/非线性对比。  
- 能参与预测评估的只有 **D1、D2、D4**（谁有金标准谁才能算 r/RMSE）。若要把「其他维度」纳入，目前指的就是 **D4**；D3 只能用于表型/聚类或 Level 3 的 encode，不能用于金标准回归或分类的数值评估。

## 5. 二分类任务：IR 是否存在、β 细胞是否有问题

你要的两个任务可以这样落地：

- **任务 1**：是否存在胰岛素抵抗（IR）  
  - 定义：用金标准 SSPG（或 HOMA-IR）定一个阈值，例如 SSPG > 某值 或 HOMA_IR > 某值 为「存在 IR」，否则为「不存在」。  
  - 做法：用 6D latent（或 si / 或 Ridge 预测的 SSPG）训练一个二分类器（如 Logistic 回归、或直接对 Ridge 预测值阈值化），在 5-fold 上报告 AUC、灵敏度/特异度等，并和「仅用 HbA1c」「仅用空腹血糖」等比。

- **任务 2**：β 细胞功能是否有问题  
  - 定义：用金标准 DI 定阈值，例如 DI < 某值 为「β 细胞功能受损」，否则为「正常」。  
  - 做法：同上，用 6D latent（或 mi / 或 Ridge 预测的 DI）做二分类，5-fold 评估并和基线比。

- **和现有结果的关系**  
  - 我们已经在做「对 DI 和 SSPG 的回归预测」；二分类是在同一预测上**加阈值**，或单独训一个分类头。  
  - 要「两个都拿最好的」：主结果继续用 **V4 的 D1+D2** 回归（SSPG r=0.65, DI r=0.51）；若你希望分类指标也最好，可以用同一 run 的 latent 或预测值做分类，并对比不同阈值、不同基线。

## 6. 若想换种方式处理 D4

- **不参与主评估**：已做——V4 用 `P1_GOLD_DATASETS=D1,D2`，主表主图不含 D4。  
- **参与训练、不参与选优**：保持 D1+D2+D4 训练，选优仍按 D1+D2 的 5-fold r；D4 仅作数据增强。  
- **按 dataset 加权**：在 Ridge 或损失里对样本按 `dataset_id` 加权（例如降低 D4 权重），减少 D4 对全局线的影响；需要改评估脚本或训练脚本。  
- **分层报告**：主表只报 D1+D2；补充材料里单独报「D4 上 r/RMSE」，并注明 D4 样本量小、DI 范围窄、仅作参考。

当前推荐仍是：**主结果以 V4（仅 D1+D2）为准**，D4 在文中说明为「另一队列/不同定义，仅作补充」，不在主表里用 D4 的留一数据集出或 5-fold 数字。
